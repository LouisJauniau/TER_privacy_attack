Attaques

  • Linkage attack : base auxiliaire avec age, sex, native-country; 50 cibles; seed 42; sans privJedAI.
  • MIA : mêmes attributs connus; out_frac=0.05, in_frac=0.05; 1000 cibles équilibrées (500 IN / 500 OUT); seed 42.
  • La MIA repose sur le dataset publié adult_mia_demo.published.csv, anonymisé ensuite avec la même configuration ARX.
Point méthodologique utile. Le protocole est propre : même logique d’attaquant pour la linkage et la MIA, même noyau de configuration ARX, et séparation claire entre dataset publié et dataset d’évaluation interne.

Interprétation. Sur une base équilibrée 500 IN / 500 OUT, une accuracy de 0.498 est pratiquement équivalente au hasard. Le rappel membre n’est que de 0.078, avec 461 faux négatifs sur 500 membres.
Pourquoi la MIA échoue ici ? Les ensembles de candidats compatibles restent énormes des deux côtés : en moyenne 4291.2 pour les membres et 4193.8 pour les non-membres. Avec les seuils min_best_score=0.5, max_compatible_candidates=100 et max_compatible_fraction=0.01, la règle de décision filtre très peu la différence entre IN et OUT.

Synthèse

  • The anonymization configuration is stable across both experiments: QI = age, sex, race, marital-status, sensitive attribute = income, k = 5, l = 2, suppression limit = 0.1.
  • In the linkage setting ARX generalized age to level 2 while leaving sex race and marital-status at level 0; 1 918 rows were suppressed (5.9%).
  • The linkage attack never achieved unique exact reidentification (0%), but the true record stayed inside the compatibility class for all 50 targets.
  • The average linkage equivalence class still contains 4617 candidates, yet the average probability assigned to the target’s true income is 0.715, which means attribute inference remains informative even without exact reidentification.
  • For MIA, the classifier is almost at chance level: accuracy = 0.498 on a balanced 500/500 split, with recall for members only 0.078.
  • The MIA candidate sets remain huge for both groups (members: 4291.2; non-members: 4193.8), which explains why the rule mostly predicts OUT.
